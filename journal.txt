PROBLÈMES RENCONTRÉS ET SOLUTIONS
----------------------------------

1. DOCKER — permission denied sur /var/run/docker.sock
   Cause   : session ouverte avant l'ajout au groupe docker
   Fix     : fermer/rouvrir la session (newgrp docker ne persiste pas)

2. CADDY — port 80 déjà occupé
   Cause   : un autre service tournait sur le port 80
   Fix     : passage sur le port 8080, puis finalement 443 (HTTPS)

3. AUTHELIA — crash au démarrage (3 erreurs de config)
   Erreur 1: domain 'localhost' invalide (doit avoir un point)
   Erreur 2: authelia_url doit être HTTPS
   Erreur 3: default_redirection_url == authelia_url interdit
   Fix     : changement de domaine localhost → llm.test, passage HTTPS

4. PERMISSIONS — fichiers authelia owned par root
   Cause   : fichiers créés initialement par Claude Code (root)
   Fix     : sudo chown -R hugo_leonardi:hugo_leonardi ~/llm-server/authelia/

COMMANDES UTILES
----------------
# Démarrer tous les services
sudo docker compose up -d

# Arrêter tous les services
sudo docker compose down

# Voir l'état des conteneurs
sudo docker compose ps

# Logs d'un service
sudo docker compose logs authelia --tail 50
sudo docker compose logs caddy --tail 50

# Redémarrer un seul service
sudo docker compose restart authelia

# Exporter le certificat CA local de Caddy
sudo docker cp caddy:/data/caddy/pki/authorities/local/root.crt ~/llm-server/caddy-local-ca.crt
